MNIST Classifier

I used principal component analysis to extract the essential features from the images and improve classification performance and efficiency by transforming the high-dimensional image data into a lower-dimensional feature vector. Implementing this involved the following steps:

  1. Splitting testing and training – I split up the PCA process for testing and training by only computing the mean and principal components during training and saving them to a separate file to be loaded for testing, to ensure consistency between the two stages.
  2. Pre-processing Techniques:
    1. Data standardisation – after experimentation, I found that standardising the data to have zero mean and unit variance yielded much better results than normalisation, as it ensures each feature contributes equally to variance computation.
    2. Adaptive Noise – I added random gaussian noise that is scaled based on the standard deviation of each training image to increase the models robustness to noise and avoid overfitting.
  3. Computing the Covariance Matrix – I calculate the covariance matrix as it captures each pixel position as a variable which is essential for PCA, as it is later used to identify the principal components
  4. Computing the Eigenvalues and eigenvectors – I calculate the Eigenvectors of the covariance matrix to identify the principal components and the Eigenvalues to identify the significance of each direction (variance). I then sort them in order of importance for later feature selection.
  5. Feature selection – I select the number of components based on how many are required to capture at least 99% of the variance as this gives a good threshold to reduce features, while still preserving useful information.
  6. Projecting the data onto the principal components – I centre and project the data onto the principal components to reduce the dimensional-space.
